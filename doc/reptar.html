definitions(definitions=None)[source]#

Add definitions of data to the file.

The base definitions are included by default. There should be no overlap of any definitions. If there is, precedence is given in order of definitions with base always being first.

Parameters

definitions (list [str], optional) – Paths to data definition YAML files to congregate. Only the name (not a path) is required for ones provided by reptar.

Notes

Reserved definition YAML files: base, md, molprop, pes, qc, sampl, solv, xtb.

Parameters

ids(group_key, entity_ids, comp_ids)[source]#

Add entity_ids and comp_ids to a group.

Parameters

  • group_key (str) – Key to the group to add IDs to (including parent).

  • entity_ids (numpy.ndarray) – A uniquely identifying integer specifying what atoms belong to which entities. Entities can be a related set of atoms, molecules, or functional group.

  • comp_ids (numpy.ndarray) – Relates entity_id to a fragment label for chemical components or species. Labels could be WAT or h2o for water, MeOH for methanol, bz for benzene, etc. There are no standardized labels for species. The index of the label is the respective entity_id.

simplify_iter_data(data)[source]#

Simplify iterative data if possible.

Attempts to simplify iterative objects by the type and/or number of elements. For example, iterative data with only one element will return just that element.

This is mainly used when adding data to a file. When parsing data we will not know if there will be multiple values of certain data (e.g., multiple single point energies) until we finish parsing. Thus, we assume there will be multiple values while parsing then simplify any iterative objects at the end.

Parameters

data (numpy.ndarray, list, tuple) – An iterative data object.

reptar.sampler module#

reptar.sampler.add_structures_to_group(source_data, source_key, dest_data, dest_key, quantity, comp_labels, structure_idxs=None, consistent_entities=None, criteria=None, z_slice=[], cutoff=[], center_structures=False, sampling_updates=False, copy_EF=False, energy_label='energy_ele', force_label='forces', write=True)[source]#

Adds randomly sampled structures to a exdir.Group.

Parameters

  • source_data (reptar.data) – A reptar data object to sample structures from. Must contain atomic_numbers, geometry, entity_ids, and comp_ids. If it does not contain a md5_structures one will be created and saved.

  • source_key (str) – Key to the desired source group.

  • dest_data (reptar.data) – A reptar data object to add sampled structures to.

  • dest_key (str) – Key to the desired destination group. If it does not exist then it will be created.

  • quantity (str or int) – Number of structures to sample from the data. For example, '100', 452, or 'all'.

  • comp_labels (tuple) – The component ID labels to include in sample structures. The order of each label does matter.

  • structure_idxs (tuple, optional) – Specific structure indices from source to sample from. Defaults to all structures.

  • consistent_entities (list [int], optional) – Molecule indices that will be in every selection. Not implemented yet. Defaults to None.

  • criteria (object, optional) – Structure criteria during the sampling procedure. Defaults to None if no criteria should be used.

  • z_slice (numpy.ndarray, optional) – Indices of the atoms to be used for the cutoff calculation. Defaults to [] is no criteria is selected or if it is not required for the selected criteria.

  • cutoff (list, optional) – Distance cutoff between the atoms selected by z_slice. Must be in the same units (e.g., Angstrom) as R. Defaults to [] if no criteria is selected or a cutoff is not desired.

  • center_structures (bool, optional) – Move the center of mass of each structure to the origin thereby centering each structure (and losing the actual coordinates of the structure). Defaults to False.

  • sampling_updates (bool, optional) – Will print something for every 100 successfully sampled structures. Defaults to False.

  • copy_EF (bool, optional) – Creates datasets for energies and forces (using energy_label and force_label) and attempts to copy data from the source if possible. If no compatible data is available it will just store numpy.NaN. Defaults to True.

  • energy_label (str, optional) – Specifies the name of the destination dataset with the desired energies. Defaults to 'energy_ele'. This should be the same as the source.

  • force_label (str, optional) – Specifies the name of the destination dataset with the desired atomic forces. Defaults to 'forces'. This should be the same as the source.

  • write (bool, optional) – Write newly sampled to Group. Defaults to True

Notes

Nothing is saved until all checks are passed to avoid irreversible changes to already existing Group.

reptar.sampler.sample_structures(source_data, source_key, quantity, comp_labels, R_prov_ids, source_R_prov_specs, Z=None, R=None, R_prov_specs=None, structure_idxs=None, criteria=None, z_slice=None, cutoff=None, center_structures=False, sampling_updates=False, copy_EF=False, E=None, F=None, energy_label='energy_ele', force_label='forces')[source]#

Randomly samples structures from a source.

If previous data are already provided (i.e., Z, R, and R_prov_specs are not None) then the sampled structures are appended to these arrays

Parameters

  • source_data (exdir.Group) – A loaded exdir.Group to sample structures from. Must contain z, r, entity_ids, and comp_ids.

  • source_key (str) – Key to the desired source group.

  • quantity (str) – Number of structures to sample from the structure set. For example, '100', '452', or even 'all'.

  • comp_labels (tuple) – The component_id labels to include in sample structures. The order of each label does matter.

  • R_prov_ids (dict {int: str}) – Structure prov IDs including any originally defined IDs and new ones being added from the source.

  • source_R_prov_specs (numpy.ndarray or None) – Structure prov specifications of the source if it was created by sampling from other sources. If the source is original, this should be None.

  • Z (numpy.ndarray, optional) – Atomic numbers of destination if they already exist. Defaults to None.

  • R (numpy.ndarray, optional) – Cartesian atomic coordinates of destination if they already exist. Defaults to None.

  • R_prov_specs (numpy.ndarray, optional) – Structure prov specifications of destination. Defaults to None.

  • structure_idxs (tuple, optional) – Source indices of structures to sample from. Defaults to sampling from all structures.

  • criteria (func, optional) – Structure criteria during the sampling procedure. Defaults to None if no criteria should be used.

  • z_slice (numpy.ndarray, optional) – Indices of the atoms to be used for the cutoff calculation. Defaults to [] is no criteria is selected or if it is not required for the selected criteria.

  • cutoff (list, optional) – Distance cutoff between the atoms selected by z_slice. Must be in the same units (e.g., Angstrom) as R. Defaults to None if no criteria is selected or a cutoff is not desired.

  • center_structures (bool, optional) – Move the center of mass of each structure to the origin thereby centering each structure (and losing the actual coordinates of the structure). Defaults to False.

  • sampling_updates (bool, optional) – Will print something for every 100 successfully sampled structures. Defaults to False.

  • copy_EF (bool, optional) – Creates datasets for energies and forces (using energy_label and force_label) and attempts to copy data from the source if possible. If no compatible data is available it will just store numpy.NaN. Defaults to True.

  • E (numpy.ndarray, optional) – Original energies, if any, to append to. Defaults to None.

  • F (numpy.ndarray, optional) – Original atomic forces, if any, to append to. Defaults to None.

  • energy_label_source (str, optional) – Specifies source dataset name containing energies to copy. Defaults to 'energy_ele'. If copy_EF is True, but there are no valid energy datasets then we create an empty array.

  • force_label_source (str, optional) – Specifies source dataset name containing atomic forces to copy. Defaults to 'forces'. If copy_EF is True, but there are no valid energy datasets then we create an empty array.

Returns

  • numpy.ndarray – Atomic coordinates of sampled structures.

  • numpy.ndarray – Atomic coordinates of sampled structures appended to any original structures.

  • numpy.ndarray or None – Energies of sampled structures appended to any original structures. None if copy_EF is False.

  • numpy.ndarray or None – Atomic forces of sampled structures appended to any original structures. None if copy_EF is False.

  • numpy.ndarray – entity_ids of the structures.

  • numpy.ndarray – R_prov_specs of the structures.
